React-Native Cli의 역할은?

사실, React-Native는 React-Native CLI와 Expo 두 가지 방법으로 개발이 가능하다.

둘은 약간의 차이가 있다.

| 항목 | 소개 | 장점 | 단점 | | ---------------- | -------------------------- | ---- | ---- | | React-Native CLI | • 기본 화면은 App.tsx이다. |

• 0.71 버전부터는 typescript가 자동적용 • 외부 라이브러리 다운로드 시 반드시 CocoaPods를 사용하여 종속성(dependency)을 설치해주어야 한다. (종속성 설치 명령어 : $ cd ios && pod install && cd..) | • 개발 환경을 구축하기가 쉽다. • 다양한 라이브러리 사용 가능 • Native 모듈 사용 가능 • 커스터마이즈 (높은 자유도) | • 빌드 시 플랫폼별 IDE가 필요 (Android Studio, Xcode Mac 필요) • 모든 라이브러리 직접 설치 | | Expo CLI | • 기본 화면은 App.js이다. 해보니까 App.tsx로 요즘은 바뀐 듯하다. | • React Natvie를 위한 기본 설정을 미리 구성 • 개발하기 어려운 기능들이 기본으로 탑재되어 있다. (앱푸시, 애플로그인 등) • 플랫폼 별 IDE가 없어도 QR코드를 통해서 해당 프로젝트를 실행 가능 | • expo에서 지원하지 않는 네이티브 라이브러리는 쓸 수 없다. • Expo 서버로 빌드할 경우 무료 사용자들은 자신의 앱 빌드를 차례를 대기해야 한다. |

나는 Expo를 써보기로 했다. Expo를 이용하면 다른 기기에서 원격으로 실행이 가능하기 때문이다.

Expo 기본 세팅

일단 필요한 패키지들을 설치하자.

npm install -g eas-cli

이제 프로젝트를 생성할 폴더를 만들어주고 해당 폴더에서 터미널을 열어 아래와 같은 명령어를 친다.

npx create-expo-app [App-Name]

이제 해당 프로젝트로 이동해 로그인을 하자.

회원 가입은 아래 사이트에서 가입해주면 된다.

Expo

cd [App-Name]
npx expo login

로그인 시에 입력하는 아이디와 비밀번호를 차례로 입력해주면 기본 세팅은 끝이 난다.

이제 위 Expo 사이트로 들어가 로그인을 해준다.

대시보드에서 All Projects > Create a project를 선택해준다.

프로젝트를 생성했다면, slug 옆에 있는 프로젝트 id를 복사하고 아래와 같은 명령어를 입력해준다.

eas init --id [복사한 ID]

이제 앱을 실행해보자.

npx expo start

여기서 a를 누르면 안드로이드 에뮬레이터가 실행되고, i를 누르면 ios 에뮬레이터가 실행된다.

또한 옵션 설명과 더불어 맨 위에 QR 코드가 하나 뜰텐데, 만약 휴대폰에 Expo Go를 설치했다면 로그인 후 카메라로 스캔해주면 원격으로 다른 기기에서 실행할 수도 있다.

추가: expo 실행 시 too many files와 같은 오류 메시지가 발견되면 아래와 같은 명령어를 입력해주자.

Error: EMFILE: too many open files - React Native CLI

brew update
brew install watchman

watchman을 설치해주면 된다.
