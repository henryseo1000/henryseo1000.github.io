Apache Tomcat 설치하기

태그: Linux, Mac 날짜: 2024년 8월 11일 파일과 미디어: https://bluexmas.tistory.com/1189, https://velog.io/@ung6860/%EA%B0%9C%EB%B0%9C%ED%99%98%EA%B2%BDUbuntu%EC%97%90-Java-%EC%84%A4%EC%B9%98%ED%95%98%EA%B8%B0

개요

이번에 Spring을 공부하고 간단한 프로젝트를 하게 되면서,

Spring 서버를 실행시키기 위해 Apache Tomcat이 필요하게 되어 이렇게 설치하게 되었습니다.

제가 정리하려고 하는 방법은 크게 2가지 입니다.

  1. 맥북에 설치하여 사용하기
  2. Raspberry PI에 설치해 사용해보기

맥북에 설치하기

M3 맥북 프로입니다.

https://tomcat.apache.org/index.html

위의 사이트에 접속해 원하는 버전을 선택하고, 다운로드하도록 하겠습니다.

저는 Tomcat 10을 사용하기로 하였습니다.

위 항목 중 tar.gz를 선택하여 다운받겠습니다.

다운로드 받은 파일의 압축을 풀어주면 아래와 같은 폴더가 나오게 됩니다.

압축을 풀어 나온 폴더를 원하는 폴더에 놓아두면 됩니다.

저의 경우, /opt 폴더에 놓아 두었습니다.

이제, 사용할 준비는 완료되었습니다.

Raspberry PI에서 설치하기

이번에는, 라즈베리 파이에 설치해보겠습니다.

먼저, java를 설치해주어야 합니다.

그 전에, java의 버전 리스트를 받아와봅시다.

sudo apt-get update

sudo apt list openjdk*

#또는 특정 버전에 대한 리스트만 받아오고 싶을 경우
sudo apt list openjdk-[버전]*

저는 설치하고 싶은 java 버전을 17로 정했기 때문에,

아래와 같이 명령어를 입력하였습니다.

sudo apt-get install openjdk-17-jdk

오류가 나지 않았다면, 아래와 같이 입력하여 java와 javac 명령어가 잘 먹는지 확인해봅시다.

java --version
javac -version

위 맥북의 과정과 같은 과정을 거치되, 위 tar.gz 파일을 설치하는 부분에서 링크 복사를 합니다.

이제 이 부분을 wget 명령어로 설치해주면 됩니다.

아래와 같이 설치해줍니다.

wget [복사한 링크 주소]

wget https://dlcdn.apache.org/tomcat/tomcat-10/v10.1.28/bin/apache-tomcat-10.1.28.tar.gz

홈 디렉토리에 잘 설치가 된 것을 볼 수 있습니다.

이제 해당 파일의 압축을 풀어주겠습니다.

아래 명령어를 입력해 압축을 풀어봅시다.

sudo tar xvf [받은 tar.gz 파일 이름] -C /usr/local

위 명령어는 /usr/local에 압축을 풀겠다는 의미입니다.

따라서 이제 /usr/local에 압축이 잘 풀렸는지 확인하겠습니다.

하위 폴더에 apache-tomcat 폴더가 잘 생성된 걸 확인할 수 있습니다.

이제 cd 명령어를 통해 아래 apache-tomcat 폴더로 이동하겠습니다.

cd apache-tomcat-10.1.28

현재는 apache-tomcat의 압축만 풀었을 뿐, 시스템의 서비스로서 등록하지는 않았습니다.

이제부터 해당 과정을 진행해보겠습니다.

먼저 tomcat.service 파일을 생성합니다.

sudo vi /etc/systemd/system/tomcat.service

위 명령어로 파일이 생성되었다면, 아래 내용을 입력해주면 됩니다.

[Unit]
Description=tomcat 10 #톰캣 버전에 맞추어 입력
After=network.target syslog.target

[Service]
Type=forking
Environment="JAVA_HOME=자바 경로"
Environment="CATALINA_HOME=톰캣 경로"
User=root
Group=root
ExecStart=톰캣경로/bin/startup.sh
ExecStop=톰캣경로/bin/shutdown.sh

[Install]
WantedBy=multi-user.target

모두 적어주고 나서는 esc 키를 누르고 저장합시다.

저는 아래와 같이 다른 블로그를 참고하여 새로 작성해주었습니다.

위와 같이 서비스와 관련된 .service를 설정해줄 경우에는 아래와 같은 규칙을 따릅니다.

  1. []의 대괄호로 구분되는 각 부분은 Section이라고 하는데, 각 Section의 이름 (Unit, Service)은 한 줄만 쓰고, 각 Section에 해당하는 내용은 줄바꿈으로 구분합니다.
  2. 각 서비스의 Section의 이름은 대소문자를 구분하며, 잘못된 Section의 이름을 사용할 경우 오류가 발생합니다.

제가 이걸 왜 아냐구요?

Unit 대문자로 썼다가 톰캣이 실행 안된다고 저항하는 바람에… 삽질 좀 했습니다.

또한 한 가지 더 :

java 경로를 작성할 경우에는 자바 경로 그대로 나오는걸 쓰지 말고 끝에 bin/java 부분은 떼고 씁니다.

JDK의 경로를 지정해 주는거지, JRE가 어디 있는지는 궁금하지 않다고 하네요… ~~이 요망한 녀석…~~

이렇게 잘 설정해주고 나면, 아래의 명령어를 순서대로 입력해줍니다.

systemctl enable tomcat.service

# start를 했을 때 에러가 뜨면 안됩니다. 에러가 뜬다면 어딘가 설정을 잘못하신 겁니다.
systemctl start tomcat.service

# start 시 에러가 뜨지 않는다면 아래 명령어 입력해 확인
systemctl status tomcat.service

위와 같이 초록색으로 **active(running)**이 뜨면 성공입니다.

만약 성공하지 못하셨다면 아래 명령어를 입력 후 :

systemctl disable tomcat.service

tomcat.service를 수정하신 뒤 다시 위 명령어를 차례대로 입력해주세요.

그렇지 않으면 제대로 반영이 되지 않으니 주의해주셔야 합니다.

이상으로 라즈베리 파이에 apache-tomcat을 설치하고 설정하는 방법을 알아보았습니다.
